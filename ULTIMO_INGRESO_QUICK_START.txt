🔧 HOW IT WORKS NOW
================================================================================

SEQUENCE:

1. User visits login page (/login)

2. User enters username and password

3. Backend checks password ✓ Correct
   └─> Sets: user.ultimo_ingreso = datetime.utcnow()
   └─> Saves: db.session.commit()
   └─> Logs: Console shows timestamp

4. Browser redirects to /buscar

5. User clicks ⚙️ Opciones button

6. JavaScript calls: fetch("/seguridad/api/usuarios")

7. Backend returns JSON with all users + ultimo_ingreso field
   Example: {"ultimo_ingreso": "18-03-2026 14:35"}

8. JavaScript renders table with "Último Ingreso" column

9. Table shows date/time in format: DD-MM-YYYY HH:MM

================================================================================
🔍 DEBUGGING TIPS
================================================================================

If "Último Ingreso" shows "-" for all users:
  1. Check you actually logged in (see terminal message)
  2. Ensure db.session.commit() succeeded
  3. Try logging in again
  4. Refresh page with Ctrl+F5

If column doesn't show at all:
  1. Check buscar.html line ~404 for <th>Último Ingreso</th>
  2. Refresh with Ctrl+F5 to clear cache
  3. Open browser console (F12) for errors

If date format is wrong:
  1. Check format in routes.py: "%d-%m-%Y %H:%M"
  2. Should show: "18-03-2026 14:35" (not "2026-03-18" or other formats)

If console is empty:
  1. Open F12 Developer Tools
  2. Make sure you're on "Console" tab
  3. Click ⚙️ Opciones button to trigger fetch
